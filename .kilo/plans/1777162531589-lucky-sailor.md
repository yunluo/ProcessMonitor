# Process Monitor 代码分析与改进计划

## 项目概述
轻量级 Windows 进程监控程序，C 语言编写，单文件实现，用于定时任务调用检查进程是否运行并在未运行时启动。

## 发现的问题

### 严重问题 (High Priority)

#### 1. 内存泄漏风险
**位置**: `process_monitor.c:87-145`

如果 `argv` 分配失败提前返回，`cmd_copy` 会泄漏。

#### 2. process_name 提取逻辑缺陷
**位置**: `process_monitor.c:500-527`

只处理 .exe 扩展名，如果 command 是 `python script.py` 或 `notepad.exe config.ini`，process_name 会为空。

#### 3. INI 解析不支持注释
**位置**: `process_monitor.c:360-408`

`get_ini_value` 函数不处理 `;` 注释。`command = "C:\path\to\app.exe" ; 这是注释` 会把注释内容当作命令的一部分。

#### 4. INI key 大小写敏感
**位置**: `process_monitor.c:386`

使用 `strncmp` 而非大小写不敏感比较。`Command` 和 `command` 会被视为不同 key。

#### 5. 命令行解析引号处理缺陷
**位置**: `process_monitor.c:111-122`

未闭合的引号会导致参数解析错误。

#### 6. section 名超长静默跳过
**位置**: `process_monitor.c:459`

如果 section 名超过 64 字符会被静默跳过，用户不会知道配置丢失。

### 中等问题 (Medium Priority)

#### 7. 魔法数字
- 行 187: `configs[50]` - 配置数组固定为 50
- 行 459: `section_total < 100` - 最大章节数限制

#### 8. 错误处理不完整
- `write_log` 如果文件打开失败直接静默返回
- `CreateDirectoryA` 的返回值未检查
- `localtime` 返回 NULL 虽有处理但不完善

#### 9. 进程启动验证缺失
**位置**: `process_monitor.c:714`

`CreateProcessA` 成功返回但进程立即退出时无法检测。

#### 10. 构建脚本与文档不符
`build_final.bat` 只生成一个版本，README 声称生成三个版本。

#### 11. 编译器路径硬编码
`D:\Program Files (x86)\i686-5.3.0-release-win32-dwarf-rt_v4-rev0\mingw32\bin\gcc.exe`

### 低优先级问题 (Low Priority)

#### 12. Windows 版本目标过时
编译时 `_WIN32_WINNT=0x0501` 强制 XP 兼容，XP 已终止支持。

#### 13. 缺少进程启动后验证
没有等待或验证进程是否真正进入运行状态。

#### 14. 全局变量线程不安全
`ini_file_path` 和 `log_file_path` 是全局变量。

#### 15. Windows API 头文件缺失兼容性
`psapi.h` 用于 `GetModuleFileNameA`，但代码中没有实际调用 Psapi 函数。

### 已验证无问题

#### strncat 使用经验证正确
**位置**: `process_monitor.c:696-698`

`strncat(full_working_dir, working_dir, sizeof(full_working_dir) - strlen(full_working_dir) - 1)` 实际上是正确的用法。

## 改进建议

### 方案 A: 修复现有问题（推荐）
- 修复所有内存泄漏
- 修复 process_name 提取逻辑
- 支持 INI 注释和 key 大小写不敏感
- 修复命令行引号解析
- 增加 section 名超长警告
- 消除魔法数字
- 改进错误处理
- 修复构建脚本

### 方案 B: 增强功能
在 A 基础上增加：
- 支持环境变量（如 `%PROGRAMFILES%`）
- 进程启动重试机制
- 配置文件语法验证
- 干运行模式（测试配置）

### 方案 C: 重构
完全重构代码：
- 模块化拆分（分离 INI 解析、进程管理、日志等）
- 添加单元测试框架
- 跨平台支持（Linux）
